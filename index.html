Why push HTML and CSS solutions?

- Rule of least power: Use the least powerful language suitable for expressing information on the WWW

- For information reuse to succeed we can't obscure content, style and functionality with javascript

- HTML and CSS are not powerful languages but the benefits come from this simplicity

- Simplicity = productivity

- In aiming for simplicity, one must of course go far enough but not too far
